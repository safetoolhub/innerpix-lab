**Rol y Contexto:**
Eres un ingeniero de software senior especializado en Python, con experiencia demostrable en visión por computadora (OpenCV, imagehash), gestión de archivos multimedia y desarrollo de aplicaciones GUI robustas con PyQt5. Has desarrollado múltiples herramientas de gestión de medios digitales con más de 10,000 archivos procesados.

**Objetivo Principal:**
Desarrollar una aplicación Python de escritorio con interfaz gráfica llamada "photokit Manager" para detectar y gestionar archivos multimedia duplicados mediante análisis de similitud perceptual, no solo por nombre de archivo.

**Alcance de Archivos:**
Cuando se menciona "fotos" en este documento, se incluyen imágenes (HEIC, JPG, JPEG, PNG, TIFF, BMP, WebP) y videos (MP4, MOV, AVI, MKV, WMV, FLV).

***

### Requisitos Funcionales de la aplicacion

**Análisis de Archivos:**

- Escanear recursivamente un directorio raíz y todos sus subdirectorios
- Procesar archivos independientemente del nombre (archivos con mismo nombre pero diferente contenido deben analizarse por contenido)
- Detectar duplicados incluso con variaciones en resolución, compresión, metadatos EXIF o ligeras modificaciones

**Detección de Duplicados:**

- Implementar hashing perceptual usando imagehash (phash, dhash, average hash) con hash_size configurable (valor recomendado: 8)
- Para imágenes: calcular similitud mediante comparación de hashes con threshold ajustable (Hamming distance)
- Para videos: extraer frames clave (primer frame, frame medio, frame final) y aplicar hashing perceptual
- Agrupar archivos similares con score de similitud visible para el usuario

***

### Requisitos Técnicos e Interfaz

**Tecnología Base:**

- Python 3.9+
- GUI: PyQt5 (preferido por robustez y Qt Designer) o Tkinter (alternativa ligera)

**Componentes de la Interfaz:**

1. **Panel de Configuración:**
    - Selector de directorio raíz con botón "Examinar"
    - Selector de carpeta para logs
    - Configuración de sensibilidad de detección (slider de 0-20 para Hamming distance threshold)
    - Filtros de formato de archivo (checkboxes por tipo)
    - Barra de progreso con indicador de archivos procesados/total
2. **Panel de Resultados:**
    - Vista de grupos de duplicados en lista o grid
    - Miniaturas/previews en color con información: ruta completa, tamaño, resolución, fecha de creación/modificación
    - Indicador visual del archivo seleccionado por defecto (mayor tamaño o mejor resolución)
    - Permitir cambiar selección manualmente con radio buttons
    - Mostrar score de similitud entre archivos del grupo
3. **Panel de Acciones:**
    - Funcionalidad "Analizar" (escaneo sin modificación)
    - Funcionalidad "Live Photos" (limpia imagenes o videos duplicados de Live Photos)
    - Funcionalidad "Duplicados HEIC" (eliminar ficheros HEIC que duplican una foto)
    - Funcionalidad "Unificar Directorio" (mover todos los archivos al directorio raiz)
    - Funcionalidad "Renombrado" (Renombra archivos a un formato estandard)
    - Funcionalidad futura "Eliminar duplicados" (eliminar imagenes duplicadas)
    - Funcionalidad futura "Categorizar por fechas" (permite crear subcarpetas para años o meses)

4. **Sistema de Renombrado (Opcional):**
    - Formato: `YYYYMMDD_HHMMSS_XXX_nnn.extension`
    - XXX: "PHOTO" o "VIDEO" según tipo de archivo
    - nnn: número secuencial de 3 dígitos si hay múltiples archivos en la misma fecha. En caso de que no haya múltiples archivos no se pone sufijo.
    - Fecha: la más antigua entre fecha de creación, modificación o EXIF DateTimeOriginal
    - Ejemplo: `20251103_PHOTO_001.jpg`, `20251103_VIDEO_001.mp4`
    - Preview del renombrado antes de aplicar

### Requisitos de Funcionamiento

**Logging Completo:**

- Archivo de log con timestamp: `duplicate_finder_YYYYMMDD_HHMMSS.log`
- Registrar: rutas analizadas, número total de archivos, grupos de duplicados detectados, decisiones del usuario (conservar/eliminar), errores, tiempo de procesamiento
- Formato estructurado para facilitar auditoría posterior

**Flujo de Trabajo Seguro:**

1. Análisis sin modificación del filesystem
2. Presentación de resultados para revisión
3. Confirmación explícita del usuario
4. Ejecución de acciones con registro detallado
5. Opción de deshacer mediante respaldo temporal (opcional pero recomendado)

**Algoritmo de Detección:**

- Usar `imagehash.dhash()` con hash_size=8 como baseline
- Comparar hashes mediante Hamming distance
- Threshold configurable (recomendado: 5-10 para duplicados exactos, 10-15 para similares)
- Para videos: extraer 3 frames y promediar similitud

**Manejo de Errores:**

- Archivos corruptos o inaccesibles: registrar en log y continuar
- Permisos insuficientes: alertar al usuario
- Timeout para archivos muy grandes (configurable)
